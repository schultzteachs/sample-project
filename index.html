Tutorial for Git and Github

Git and Github Tutorial Part 1 by Dave Gray

Commands for local Git

git init

git status

git add .

git commit -m "message"

Commands for Github

git remote add origin [repository URL]

git push -u origin main

Set up user information for Git

You might get the error: "src refspec main does not match any". In this case, you need to use git config to set your user information.

git config --global user.name "Your Name"

git config --global user.email "you@example.com"

Then try committing and pushing again.

git push (pushes local commits to Github)

Workflow generally is:

git add .

(stages all changes)

git commit -m "comment message"

(commits staged changes with a message)

git push

(pushes local commits to Github)

git status

(shows the status of changes)

You should get something that looks like this:

$ git status On branch main Your branch is up to date with 'origin/main'. nothing to commit, working tree clean

Part 2 of series:

Part 2

Use git status to check status

If a file has been "staged" after doing git add, it will show up in the "Changes to be committed" section of git status.

If you need to unstage a file, you use git restore to unstage a file

Use git log and the git log --oneline command to view commit history

If a comment needs to be edited, use git commit -v --amend

This will open up a text editor with the previous commit message, allowing you to make changes.

When done with that comment edit, use ":wq" to save and exit the editor.

Manipulating Branches

Use git branch --list to create a new branch

Use git checkout [branch-name] to switch to that branch

Use git merge [branch-name] to merge that branch into the current branch

Example, to get a new branch merged into the master branch, you would use git checkout master followed by git merge [branch-name]

Use git branch -d [branch-name] to delete a branch

Make sure you are checked out to the branch you want to delete before deleting it.

Use git branch -m [old-branch-name] [new-branch-name] to rename a branch

Reviewing Changes and Branching from those

git log --oneline

git checkout "short ID"

This puts you in a "detached HEAD" state, meaning you are not on any branch.

To create a new branch from this state, use git checkout -b [new-branch-name]

Then you can make changes and commit them to this new branch.

Reverting Changes

To revert changes from a specific commit, use git revert [commit-ID]

This will create a new commit that undoes the changes from the specified commit.

Reseting Changes

To reset to a specific commit, use git reset --hard [commit-ID]

This will move the HEAD to the specified commit and discard all changes after that commit

This is more destructive than git revert, as it changes the commit history.
